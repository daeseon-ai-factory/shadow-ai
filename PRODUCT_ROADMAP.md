## § 1. NORTH STAR (변경 금지)

**한 줄 미션:**
> 개발자/지식노동자가 YouTube 영상에서 직접 구간을 잘라 라이브러리에 모으고, Anki처럼 자동 복습하면서 앱 안에서 AI 설명까지 받는 영어 쉐도잉 도구.

**5가지 가치 (지키는 것):**
1. 사용자 자유 구간 선택 (자동 분할 강제 X)
2. 평생 복습 가능한 본인 라이브러리 (일회성 콘텐츠 소비 X)
3. 앱 안에서 AI 설명 (ChatGPT 다른 탭 안 가도 됨)
4. 마찰 제거 = 차별화
5. 운영 비용 낮음 (1000명까지 거의 무료)

**11가지 OUT-OF-SCOPE (안 하는 것):**

| 항목 | 이유 |
|------|------|
| ❌ 발음 점수 / AI 평가 | 자동 채점은 학습자의 자가 판단을 막음. 비교는 사용자가 |
| ❌ 프로소디 마크업 | 시각화 과잉, 학습 본질 흐림 |
| ❌ 직독직해 표시 (시스템 제공) | 사용자 노트는 OK (본인 데이터). 시스템이 강제하면 사고 차단 |
| ❌ 사용자간 공유 / 소셜 | v0 우선순위 X. 보안/모더레이션 비용 큼 |
| ❌ 모바일 네이티브 앱 | 웹으로 충분. RN/Flutter 도입은 별도 사업 결정 |
| ❌ 결제 / Stripe | v0는 단일 사용자/지인용 |
| ❌ 영어 외 언어 | 명확한 사용자 정의 (개발자 영어) |
| ❌ 개발자 외 직종 큐레이션 | 페르소나 분산 회피 |
| ❌ Whisper STT / 자체 STT | 비용 폭발 가능 + 개발자 영상은 99% 자막 있음 |
| ❌ 자체 음성 분석 | "자동 평가" 카테고리 전체 OUT |
| ❌ 다국어 UI | 한국어 단일 |

**새 기능 제안 체크리스트:**
1. 5가지 가치 중 하나를 강화하는가?
2. 11가지 OUT-OF-SCOPE에 해당하지 않는가?
3. 1주 안에 진짜 작동하는 버전 가능한가?

**→ 세 가지 다 YES면 추가 검토. 하나라도 NO면 메모만 하고 보류.**

## § 2. 현재 상태 (v0.2.0, main 브랜치)

### ✅ 만들어진 기능 (작동 확인됨)

| 영역 | 기능 |
|------|------|
| 인증 | 회원가입 / 로그인 / 프로필 수정 / 비밀번호 변경 / JWT 24h |
| YouTube | URL 임포트 (watch?v=, youtu.be, embed, shorts, live) · oEmbed 메타데이터 · **yt-dlp 자막** · 같은 URL 재임포트 = 자가 회복 |
| 자막 | 원본 segments + **문장 단위 자동 병합** (gap < 300ms + 마침표 경계) |
| 클립 | 구간 저장 · 자동 자막 슬라이스 · 이름/태그/자막/노트 PATCH · 정렬(최신/오래된/이름/길이) · 검색 · 태그 자동완성 · 삭제 |
| 클립 선택 UX | "시작/끝" 버튼 + **자막 클릭 두 번** (1번=시작, 2번=끝, 시각화 배지) |
| 라이브러리 | 카드 그리드 · 썸네일 · JSON 내보내기 · 사용자별 격리 |
| 클립 플레이어 | 무한 반복 · 0.5x~1.5x 속도 · 큰 자막 · 키보드 단축키 (`Space/R/L/,/./0/?`) |
| AI 분석 (코드) | Claude Haiku 4.5 클라이언트 · 프롬프트 캐시 · 비동기 (트랜잭션 밖) · 3초 폴링 UI · "다시 분석" 버튼 |
| 녹음 + A/B | MediaRecorder · multipart 업로드 · 시간순 누적 · "원본 → 본인" 재생 · 파일 정리 afterCommit |
| 복습 (SRS) | SM-2 알고리즘 · Again/Hard/Good/Easy 4버튼 · 키보드 1~4 · 큐 자동 생성 · 연속 일수 위젯 |
| 큐레이션 | "개발자 추천 영상" 컬렉션 · curated-videos.yml 시드 (현재 3개 영상) |
| 클립 노트 | 사용자 메모 (직독직해/연관 단어 등 자유) |
| 설정 | 프로필 페이지 |
| 보안 | CORS env-driven · 분당 IP당 20회 로그인 rate limit · constant-time login |
| 인프라 | Docker (PostgreSQL + 백엔드) · GitHub Actions CI · 종합 스모크 스크립트 · Vitest 11개 · Playwright 13개 (자막 추가) |
| 단축키 도움말 | `?` 키 다이얼로그 |

### 🔌 만들어졌지만 비활성 (스위치만 켜면 됨)

| 항목 | 켜는 법 |
|------|---------|
| **AI 분석 응답** | `ANTHROPIC_API_KEY` 환경변수 + 백엔드 재시작. 코드는 다 있음. 키 없으면 클립 만들 때마다 분석이 FAILED 상태로 끝남 |
| **큐레이션 확장** | `backend/src/main/resources/curated-videos.yml`에 검증된 영상 URL 추가 (현재 4개, 30개 목표) |
| **실제 배포** | `DEPLOY.md` 가이드대로 Vercel + Railway. 코드 다 준비됨 |

### 🐛 알려진 한계

- YouTube 자막 추출은 `yt-dlp` 의존 (백엔드 호스트에 설치 필요). Docker 이미지에는 yet 미포함.
- Shorts URL 임포트는 됨 / 화면 세로 비율 분기는 안 됨 (현재 16:9 고정)
- 클립 검색은 `LIKE '%q%'`. 10k 클립 넘어가면 느려짐 (tsvector + GIN 필요)
- JWT 만료까지 강제 로그아웃 불가 (비밀번호 바꿔도 기존 토큰 살아있음)
- Rate limit이 노드 메모리 → 다중 노드 배포 시 Redis 필요
