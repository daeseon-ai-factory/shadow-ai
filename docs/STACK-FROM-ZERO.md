# 기술 스택 제로부터 — "이게 뭐고 왜 썼나 + 트레이드오프"

> 전제: **용어 아무것도 모른다고 치고** 다 풀어 쓴다. 비유 → 정의 → 왜 이걸 → 대안/트레이드오프 → 면접에서 이렇게 답해.
> 이거 먼저 떼고 → `STUDY-GUIDE.md`(코드 딥다이브)로.
> 면접관은 스택마다 딱 2개 물어: **"이거 왜 썼어?"** + **"안 쓰면 뭐 써? 차이는?"**. 그 답을 여기서 만든다.

---

## 0. 큰 그림 — 앱은 3덩어리다
1. **프론트엔드(Frontend)** = 사용자가 보는 화면. (우리: 모바일 앱 + 웹사이트)
2. **백엔드(Backend)** = 화면 뒤에서 데이터·로직 처리하는 서버. (우리: Java 서버)
3. **인프라(Infra)** = 백엔드가 돌아가는 컴퓨터/네트워크. (우리: 네이버 클라우드 서버 1대)

흐름: `사용자 폰` → (인터넷) → `백엔드 서버` → `데이터베이스`. 백엔드가 중간에서 "로그인 맞아?",
"이 클립 분석해줘", "복습 카드 뭐 줄까?"를 판단함.

- **API** = 프론트와 백엔드가 대화하는 약속. "이 주소로 이렇게 물어보면 이렇게 답해줄게"의 규격.
  예: `POST /api/clips` = "클립 만들어줘". (REST API = 그 약속을 HTTP 주소+동사로 표현하는 흔한 방식)

## 1. 언어 — Java 21 (백엔드) / TypeScript (프론트)

**Java가 뭐냐:** 백엔드 짜는 프로그래밍 언어. 크고 오래됐고 회사들이 많이 씀(은행·대기업).
**왜 Java:** 타입이 엄격(컴파일 때 실수 잡힘) + 생태계 거대 + 채용시장 큼. (우리 `CLAUDE.md`가 Java 21로 못박음)
**대안/트레이드오프:**
- *Node.js(JavaScript)*: 가볍고 빠르게 만듦, 프론트랑 언어 통일. 근데 타입 느슨·대규모에서 관리 어려움.
- *Python(Django)*: 빠른 개발. 근데 런타임 느리고 타입 약함.
- → Java는 **"안정·엄격·채용 큼" 대신 "코드 길고 무겁다"** 가 트레이드오프.

**TypeScript가 뭐냐:** JavaScript(브라우저/앱 언어)에 **타입을 붙인 것**. (JS = 타입 없어서 실수 잘 남 → TS가 컴파일 때 잡아줌)
**왜:** 프론트는 JS 말곤 선택지가 거의 없고, TS는 그 JS를 안전하게.
- *용어*: **타입(type)** = "이 변수는 숫자다/문자다" 같은 약속. 안 맞으면 에러 → 버그 미리 잡음.
- *용어*: **컴파일(compile)** = 사람이 쓴 코드를 컴퓨터가 실행할 형태로 변환. 이때 타입 검사함.

## 2. 백엔드 프레임워크 — Spring Boot

**프레임워크가 뭐냐:** 매번 바닥부터 안 짜게 *뼈대+도구 묶음*을 미리 준 것. (집 지을 때 골조+배관 미리 깔린 거)
**Spring Boot가 뭐냐:** Java로 백엔드(웹서버·DB연결·인증) 만들 때 *표준* 프레임워크.
**왜:** Java 백엔드 = 사실상 Spring이 업계 표준. 채용 공고에 제일 많이 뜸. DB·보안·API 다 붙어있음.
**대안/트레이드오프:**
- *Node Express/NestJS*: 가볍고 빠름. 근데 보안·DB 등 직접 더 붙여야.
- → Spring은 **"다 들어있고 표준" 대신 "무겁고 시작 느림·러닝커브"**.
- *용어*: **의존성 주입(DI, Dependency Injection)** = 객체가 필요한 부품을 *직접 안 만들고 주입받음* → 갈아끼우기·테스트 쉬움. (Spring의 핵심. `CompositeAiClient`가 여러 Client를 주입받는 게 이거)

## 3. 빌드 도구 — Gradle

**뭐냐:** 흩어진 코드+라이브러리를 모아 *실행 가능한 하나로 묶는*(빌드) 도구. 라이브러리 다운로드도 함.
**왜 Gradle:** Java 진영 표준 빌드도구(Maven과 양대산맥). 우린 Kotlin DSL(설정을 코드로) 씀.
**대안:** *Maven*(XML 설정, 더 옛날·장황). → Gradle은 "유연·빠름" 대신 "가끔 복잡".
- *용어*: **라이브러리/의존성(dependency)** = 남이 만든 코드 가져다 씀. (`build.gradle.kts`에 목록)

## 4. 데이터베이스 — PostgreSQL + JPA/Hibernate

**데이터베이스(DB)가 뭐냐:** 데이터를 *영구 저장*하는 곳. (앱 끄면 메모리는 날아감 → DB는 남음) 유저·클립·복습기록 다 여기.
**PostgreSQL가 뭐냐:** 가장 많이 쓰는 오픈소스 *관계형* DB. (표=테이블, 행=레코드. 엑셀 같은 구조)
**왜 Postgres:** 무료·강력·신뢰성·기능 풍부. 사실상 "고민 없이 고르는" 기본값.
**대안/트레이드오프:**
- *MySQL*: 비슷하게 인기. Postgres가 기능·표준 준수 더 강함.
- *MongoDB(NoSQL)*: 구조 자유로움. 근데 관계(유저-클립-복습)가 많은 우린 관계형이 맞음.
- → Postgres = "관계 데이터에 강·안정" 대신 "스키마 미리 설계 필요".

**JPA / Hibernate가 뭐냐:** **ORM**(Object-Relational Mapping). Java 객체 ↔ DB 표를 *자동 번역*.
SQL 직접 안 쓰고 `clipRepository.save(clip)` 하면 알아서 INSERT.
**왜:** SQL 노가다 줄임, 객체로 다룸. **트레이드오프:** 편한 대신 *느린 쿼리/N+1 문제* 숨음 → 가끔 직접 SQL 필요(우리 `ClipRepository`에 native query 있음).
- *용어*: **스키마(schema)** = 표의 구조 설계(어떤 컬럼·타입). **마이그레이션(migration)** = 스키마 바꾸는 버전관리된 변경.

## 5. 인증 — JWT

**인증(Authentication)이 뭐냐:** "너 진짜 그 사람 맞아?" 확인. (로그인)
**문제:** HTTP는 *기억을 못 함*(stateless). 매 요청마다 "나 로그인했어"를 어떻게 증명?
**JWT(JSON Web Token)가 뭐냐:** 로그인 성공하면 서버가 *서명한 출입증(토큰)*을 줌. 이후 요청마다 그걸 첨부 → 서버가 서명 검증 → 통과. (놀이공원 손목밴드. 위조하면 서명 안 맞아서 걸림)
**왜 JWT:** 서버가 세션을 *저장 안 해도* 됨(토큰 자체에 정보+서명). 확장 쉬움.
**대안/트레이드오프:**
- *세션(쿠키)*: 서버가 로그인 상태를 메모리/DB에 저장. 즉시 무효화 쉬움. 근데 서버 여러 대면 공유 골치.
- → JWT = "서버 저장 불필요·확장 쉬움" 대신 **"발급 후 강제 만료(로그아웃) 어려움"**.
- *우리 코드*: `auth/security/JwtTokenProvider`(발급/검증), `JwtAuthenticationFilter`(매 요청 검사).
- *용어*: **해시(hash)** = 비번을 *되돌릴 수 없게* 뭉갠 값(원본 저장 X, 비교만). **필터(filter)** = 요청이 본 로직 닿기 *전에* 가로채는 관문(여기서 토큰 검사·레이트리밋).
- *용어*: **레이트리밋(rate limit)** = "1분에 N번까지만" 횟수 제한(무차별 로그인 공격 차단). 우리 `AuthRateLimitFilter`.

## 6. AI 통합 — Gemini / OpenAI / Claude (멀티 프로바이더)

**뭐냐:** 클립 분석·영작 채점을 *AI 모델*에 시킴. 우린 한 곳 말고 *3곳*을 순서대로 폴백.
**왜 멀티:** 한 곳 = **SPOF**(Single Point Of Failure, 단일 장애점) — 거기 죽으면/막히면 전체 정지. 여러 곳 두면 하나 죽어도 다음으로.
**왜 Gemini 기본:** *무료 티어*가 넉넉 = 비용 0(개인 프로젝트엔 큼).
**트레이드오프:** 여러 프로바이더 = 응답 포맷 제각각 → *정규화 코드* 필요(각 `~Client.java`). 복잡도 ↑ 대신 안정성·비용 ↑.
- *우리 코드*: `analysis/infrastructure/CompositeAiClient.java`(93줄, 핵심), `AiAnalysisClient`(인터페이스), `application.yml`의 `ai.order`.
- *용어*: **STT(Speech-To-Text)** = 음성 → 글자 변환(말하기 채점에 OpenAI Whisper 씀). **프롬프트(prompt)** = AI에게 주는 지시문(`~Prompt.java`).

## 7. 유튜브 자막 추출 — yt-dlp + POToken + deno

**문제(면접 단골):** 데이터센터 서버 IP에서 유튜브 자막 요청 → **HTTP 200인데 0바이트**(에러도 아닌 *침묵 실패*. 제일 디버깅 어려운 종류).
**원인:** 유튜브가 "이거 봇 아냐?" 게이트. 정상 클라이언트 증명 토큰(**POToken**)을 요구.
**해결:**
- **yt-dlp** = 유튜브 다운로드/자막 추출 오픈소스 도구.
- **POToken provider(bgutil)** = 그 토큰을 *생성*해주는 작은 Node 서버(우리 컨테이너 안 4416 포트).
- **deno** = yt-dlp가 요즘 추출에 필요로 하는 JS 런타임. 없으면 추출 자체가 안 됨.
- *우리 코드/설정*: `backend/Dockerfile`(이것들 왜 한 컨테이너에 같이 넣었는지 주석), `video/application/VideoImportService.java`(디바이스 자막 → 백엔드 폴백 순서).
- *왜 한 컨테이너에:* 사이드카(옆 컨테이너) 네트워킹이 깨질 여지를 없애려고 같은 컨테이너 안 localhost로.

## 8. 모바일 앱 — React Native + Expo (+ TypeScript)

**React Native(RN)가 뭐냐:** *하나의 코드*로 iOS+안드 앱 둘 다 만드는 프레임워크(JS/TS로). (네이티브 2번 안 짬)
**Expo가 뭐냐:** RN을 *훨씬 쉽게* 쓰게 해주는 도구모음(빌드·카메라·보안저장 등 미리 포장).
**왜:** 혼자서 iOS+안드 둘 다 빠르게. 네이티브(Swift/Kotlin) 각각 짜면 2배 일.
**트레이드오프:** 편하고 빠른 대신 *극한 성능·최신 네이티브 기능*은 한계. (우리 앱엔 충분)
- *용어*: **네이티브(native)** = 그 OS 전용 언어로 짠 진짜 앱(iOS=Swift, 안드=Kotlin).
- *우리가 쓴 부속들:*
  - **expo-router** = 화면 이동(라우팅)을 *폴더 구조*로. `app/today.tsx` = `/today` 화면.
  - **TanStack Query** = 서버 데이터 가져오기·캐싱·로딩/에러 상태를 자동 관리. (직접 fetch 노가다 줄임)
  - **Zustand** = 앱 전역 상태(로그인 토큰 등)를 간단히 저장/공유. (*상태관리 라이브러리*)
  - **expo-secure-store** = 토큰을 폰 *암호화 저장소*(키체인)에 안전 보관.
- *용어*: **상태(state)** = 화면이 기억하는 값(입력칸 내용, 로그인 여부 등). **캐싱(cache)** = 한 번 받은 걸 저장해 또 안 받게.

## 9. 웹사이트 — Next.js

**뭐냐:** React(웹 UI 라이브러리) 기반의 *풀스택 웹 프레임워크*. 마케팅 페이지+블로그(daeseon.ai).
**왜:** React 표준급, SEO(검색노출)·서버렌더링 잘 됨, Vercel 배포 쉬움.
**대안:** 순수 React(SPA) — SEO 약함. → Next는 "SEO·렌더링 강" 대신 "더 복잡".
- *우리*: `next-intl`(다국어 en/ko), Tailwind(CSS를 클래스로 빠르게), Vercel 호스팅.
- *용어*: **SSR(서버사이드 렌더링)** = 화면을 *서버에서 미리 그려* 보냄(빠른 첫 화면·SEO). **SPA** = 브라우저에서 다 그림(첫 로딩 후 빠름·SEO 약함).

## 10. 컨테이너 — Docker + Docker Compose

**Docker가 뭐냐:** 앱+그 실행환경(OS·라이브러리)을 *상자 하나로 통째 포장*. "내 컴퓨터선 되는데요" 문제 해결 — 어디서든 같게 돔.
**Docker Compose가 뭐냐:** 여러 컨테이너(백엔드+DB+Caddy+토큰provider)를 *파일 하나로* 같이 띄움.
**왜:** 서버에 똑같이 재현·배포 쉬움. 우린 박스 1대에 compose로 다 띄움.
**트레이드오프:** 편한 대신 *약간의 오버헤드*·러닝커브.
- *우리*: `infrastructure/ncp/box/docker-compose.yml`(backend+postgres+caddy+pot-provider), `backend/Dockerfile`(백엔드 상자 레시피).
- *용어*: **이미지(image)** = 상자 *설계도/스냅샷*. **컨테이너(container)** = 그 이미지로 실제 돌아가는 *인스턴스*.

## 11. 리버스 프록시 + HTTPS — Caddy

**리버스 프록시가 뭐냐:** 바깥 요청을 *제일 앞에서 받아* 뒤의 백엔드로 넘기는 문지기. 주소(도메인)·보안·라우팅 담당.
**Caddy가 뭐냐:** 그 문지기 서버. **HTTPS(TLS) 인증서를 자동**으로 발급·갱신해주는 게 강점.
**왜:** 설정 쉽고 TLS 자동(직접 인증서 관리 안 해도 됨).
**대안:** *Nginx*(더 강력·널리 쓰임, 근데 TLS 수동·설정 장황). → Caddy = "쉬움·자동 TLS" 대신 "Nginx보다 덜 범용".
- *용어*: **HTTPS/TLS** = 통신 암호화(자물쇠). **인증서(certificate)** = "이 도메인 진짜다" 증명서(Let's Encrypt가 무료 발급). **도메인(domain)** = `mimi.daeseon.ai` 같은 주소.
- *우리*: `infrastructure/ncp/box/Caddyfile`.

## 12. 클라우드 + 코드형 인프라 — NCP + Terraform

**클라우드가 뭐냐:** 남의 데이터센터 컴퓨터를 *빌려* 서버 돌림. (직접 컴퓨터 안 사도 됨)
**NCP(Naver Cloud Platform):** 네이버의 클라우드. **왜:** 한국 타깃 → 한국 데이터센터(서울)면 빠르고(레이턴시↓) 쌌음.
**Terraform가 뭐냐:** **IaC(Infrastructure as Code)** — 서버·네트워크를 *클릭이 아니라 코드로* 정의. 파일 실행하면 인프라가 그대로 생성됨.
**왜:** 재현 가능·버전관리·실수 줄임("어디 뭘 클릭했더라" 방지).
**트레이드오프:** 처음 배우기 빡셈 대신 *반복·이전·기록*에 강함.
- *우리*: `infrastructure/ncp/*.tf`. **이게 마이그레이션 스토리의 핵심** — AWS→NCP를 코드로.
- *용어*: **레이턴시(latency)** = 요청-응답 지연시간(ms). 멀수록 큼(토론토→한국 ~180ms).
- *용어*: **리전(region)** = 데이터센터 위치(서울/도쿄/버지니아…). 가까울수록 빠름.

## 13. DNS + 스토리지 — Cloudflare + R2

**DNS가 뭐냐:** 도메인(`mimi.daeseon.ai`)을 *서버 IP주소*로 바꿔주는 전화번호부. **Cloudflare** = 그 DNS+보안+CDN 서비스.
**왜:** 무료·빠름·DDoS 방어. 도메인 관리 여기서.
**R2가 뭐냐:** Cloudflare의 *파일 저장소*(S3 호환). 우린 DB 백업 보관에 씀.
- *용어*: **DNS** = 도메인↔IP 변환. **CDN** = 정적 파일을 전세계 가까운 곳에 캐시해 빨리 전달. **S3** = 아마존의 대표 파일저장소(R2가 그 호환). **백업(backup)** = 데이터 사본(망가지면 복구).

## 14. 모노레포 + 배포 도구

**모노레포(monorepo)가 뭐냐:** 백엔드·웹·모바일·공유코드를 *저장소 하나*에 같이 둠.
**왜:** 공유 타입/로직(`packages/core`)을 셋이 같이 씀 → 중복 제거·일관성.
- **EAS(Expo Application Services)** = Expo의 *클라우드 빌드/제출* 서비스(앱을 그쪽 서버가 빌드해 TestFlight로).
- **App Store Connect / TestFlight** = 애플의 앱 배포 관리 / 베타 테스트 도구.
- *용어*: **npm workspaces** = 모노레포 안 여러 패키지를 묶어 관리하는 방식.

## 📎 한눈 용어 사전 (모르면 여기)
- **API** 프론트-백엔드 대화 규격 · **REST** 그걸 HTTP 주소+동사로 · **HTTP** 웹 통신 규칙(GET 읽기/POST 쓰기)
- **stateless** 서버가 *기억 안 함*(매 요청 독립) · **SPOF** 단일 장애점 · **SSR/SPA** 서버렌더/브라우저렌더

- **JWT** 서명된 출입증 토큰 · **hash** 되돌릴 수 없는 뭉갬 · **filter** 요청 가로채는 관문 · **rate limit** 횟수 제한
- **container/image** 포장상자/설계도 · **reverse proxy** 앞단 문지기 · **TLS/HTTPS** 암호화 · **certificate** 도메인 증명서
- **IaC** 인프라를 코드로 · **latency** 지연(ms) · **region** 데이터센터 위치 · **DNS** 도메인↔IP · **CDN** 전세계 캐시 · **S3/R2** 파일저장소
- **state/cache** 화면 기억값/재사용 저장 · **STT** 음성→글자 · **prompt** AI 지시문
